Scripts support repérés dans .github/workflows

Deux fichiers sont rangés dans le dossier des workflows mais ne sont pas eux-mêmes des workflows YAML :

  • facebook_publisher.py : script support qui lit une bibliothèque JSON, choisit le prochain post, génère un payload Make et marque le post comme publié.
  • README_INSTALLATION.md : documentation d’installation pour la publication automatique Facebook / Instagram.

Ce que font les jobs, en clair

  • facebook_auto : pousse du contenu social de manière programmée.
  • remote_job_manual : exécute un mini-job distant et expose un état web.
  • seo_tracker : fabrique un rapport SEO mensuel publié sur le site.
  • update_videos : rafraîchit les données YouTube exploitées par le site.

Publication Phots&Videos — résumé technique

  • Le webhook reçoit au minimum title, location, description, mode, promo, mediaType et les flags de publication.
  • Les photos partent vers Facebook en lot et vers Instagram sous forme de carrousel carré 1080.
  • Les vidéos partent vers Instagram Reel, Facebook UploadVideo et YouTube UploadVideo.
  • Le module YouTube ajoute ensuite la vidéo à une playlist dédiée.

post préparé → Facebook — résumé technique

  • Le scénario lit le JSON éditorial depuis le site.
  • Il publie sur la page Facebook LCDMH.
  • Il peut aussi créer un board Pinterest correspondant à la catégorie si besoin, puis créer un pin.
  • Il sert de pont entre la bibliothèque éditoriale du site et les plateformes sociales.

Page live / journal du soir

La prochaine brique logique est une page site qui regroupe les entrées du jour et les publie le soir, par exemple à 18h00. Elle doit rester simple : titre, lieu, commentaire, vidéo/photo, lien YouTube / Facebook / Instagram.

Page pilier road trip

En parallèle, le site garde des pages plus riches : road trip en série, hubs régionaux, roadbooks PDF, GPX, QR codes, playlists et FAQ. Le journal du soir alimente ensuite cette page pilier.

Points de vigilance
Important : un webhook Make apparaît exposé en clair dans certains fichiers fournis pendant l’audit. Il faut le considérer comme compromis, le régénérer dans Make, puis remplacer l’ancienne URL partout où elle est stockée.
SujetConstatAction recommandée
Webhook MakeVisible en clair dans un script support et dans certaines pages.Régénérer le webhook et ne garder que des versions masquées dans la doc.
Langue YouTubeLe flux YouTube publie bien, mais la langue part encore en anglais par défaut.Ajouter defaultLanguage=fr et defaultAudioLanguage=fr dans le mapping Make / YouTube.
Dossier workflowsDes scripts Python / docs sont rangés dans .github/workflows.Séparer clairement workflows YAML et scripts support dans le repo.
Audit repo incompletLes dossiers automation/ et certaines zones SEO n’ont pas encore été lus en détail.Compléter le sparse-checkout pour cartographier tout le reste.
Captions YouTubeL’API officielle fonctionne mais bute sur le quota quotidien.Traiter par lots, reprendre après reset quota, conserver les transcripts comme source site.
